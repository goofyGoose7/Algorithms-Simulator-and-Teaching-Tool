## üå∏ Advice for A-Level CS Students
1. **REMEMBER this is MY project - please come up with your own idea!**
2.  MAKE YOUR PROJECT FEASIBLE IN THE TIME. I procrastinated and started properly coding mine during my Christmas holiday in Yr 13. I regretted it. Writing up and testing takes longer than you would expect so account for that if you want to be done with enough time to revise for your actual exams.
3. Stick to the code that you learn in the course - I chose to literally use Bubble Sort and Dijkstra's, which we study in the course, so that guaranteed that I would get those marks. If I were to go back, I would not have tried to code Simplex from scratch...
4. Justify EVERYTHING in your documentation to get those marks.
5. If you are running out of time, remember that nobody really reads everything you write.
6. Document as you code.
7. Label your code as you go so you can reference it in your testing tables.
8. In the design section, just convert some of your code into pseudocode once you have an idea of what you are doing. Just make it look like pseudocode.
9. It's okay to leave code that doesn't work, so long as you have documented it. Honestly it's good if it isn't 100% perfect since then you have stuff to write about in your limitations section of your evaluation
